# Clara AI Agent Configuration Assignment

## Overview

This project implements a configuration pipeline for generating and updating an AI answering agent based on customer conversations.

The system processes two types of inputs:

1. A **demo call transcript** used to create the initial configuration.
2. A **customer onboarding call** used to update and confirm configuration details.

The output of the system is a structured **AI agent specification** that can be used to configure the voice assistant.

---

# Pipeline Architecture

The project consists of two pipelines.

---

## Pipeline A — Demo Agent Creation

The demo transcript is used to generate the first version of the agent configuration.


Demo Transcript
↓
Configuration Extraction
↓
account_memo_v1.json
↓
generate_agent.py
↓
agent_spec_v1.json


This stage produces an **initial exploratory configuration**.

---

## Pipeline B — Onboarding Configuration Update

The onboarding call updates the configuration with confirmed operational information.


Onboarding Audio
↓
transcribe_audio.py
↓
onboarding_transcript.txt
↓
update_from_onboarding.py
↓
account_memo_v2.json
↓
changes.json
↓
generate_agent.py
↓
agent_spec_v2.json


This stage updates the configuration with **real business rules extracted from the onboarding call**.

---

# Key Components

## 1. Audio Transcription

File:


scripts/transcribe_audio.py


Uses the **Whisper speech-to-text model** to convert onboarding audio into text.

---

## 2. Configuration Extraction

File:


scripts/update_from_onboarding.py


This script parses the onboarding transcript and updates structured configuration fields including:

- company name
- supported services
- service pricing
- emergency policy
- after-hours behavior
- exception clients

All configuration updates are recorded in:


outputs/changes.json


This file tracks **what changed between v1 and v2 of the configuration**.

---

## 3. Agent Specification Generation

File:


scripts/generate_agent.py


This script converts the configuration memo into a deployable **AI agent specification**.

Outputs generated:


agent_spec_v1.json
agent_spec_v2.json

# Design Decisions

- Configuration stored in **structured JSON** for flexibility.
- Onboarding updates implemented using **rule-based transcript parsing**.
- Configuration changes tracked using **changes.json**.
- Agent prompt generation separated from configuration extraction.

This structure makes the system easy to extend with additional extraction rules or configuration parameters.
